# Tipos de testes
 - Teste de integração => Valida componentes isolados.
   - Validam componentes e suas respectivas respostas
 - Teste End to End => Valida o que acontece após sua ação.
   - validam o fluxo do usuário e o que a sua iteração alterou no estado da entidade.
 - Unitário => Serve para validação do seu código. 
   - Validam como o nosso código se comporta sem depender de qualquer iteração externa e geralmente são usados para testar funções individuais em contraste à integração e e2e que testam o fluxos completos
